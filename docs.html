CorePulse — Documentation

This single-page docs explains how to include CorePulse, configure data output, and ideas for dashboards.

Quick Install

npm (for bundlers):

npm install corepulse

CDN / Drop-in: copy corepulse.js to your site and include it with a script tag.

<script src="corepulse.js"></script>

Basic Usage

Initialize and start collecting metrics. The SDK logs to console by default.

CorePulse.init({ interval: 2000, track: ['fps','memory','network'] });
CorePulse.onData = function(stats){
  // stats: { fps, memory, network }
  console.log('metrics', stats);
};
CorePulse.start();

Receive & Forward Data

To handle metrics yourself, set CorePulse.onData. To automatically POST metrics to a server use CorePulse.sendTo(url):

CorePulse.sendTo('https://your-server.example.com/metrics');

On the server, accept JSON POSTs: { ts: 12345, data: { fps:..., memory:... } }. You can stream to a WebSocket to power a live dashboard.

Node & Electron

The provided browser SDK works in Electron renderer processes. For Node usage, wrap the metrics collection (FPS replaced with event-loop checks) and call the same onData interface for compatibility.

Ideas for Dashboards

  • Build a WebSocket server that broadcasts incoming metric snapshots to connected dashboards.
  • Persist snapshots in a time-series DB (InfluxDB, Timescale) for historical analysis.
  • Integrate Chart.js or Recharts to visualize live data and trends.

Troubleshooting

  • No memory data: Browser may not expose performance.memory; this is expected on some browsers.
  • No FPS: If requestAnimationFrame is not available, FPS will not collect (server-side usage).
